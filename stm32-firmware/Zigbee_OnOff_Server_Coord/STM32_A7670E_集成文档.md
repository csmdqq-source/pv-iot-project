## 📋 项目概述

本文档记录了STM32WB5MM开发板与A7670E 4G通信模块的集成过程，实现从有线Python中转到无线直连MQTT的架构升级。

### 项目目标
- ✅ USART1回环测试（已完成）
- ✅ LPUART1回环测试（已完成）
- ✅ LPUART1与A7670E透传通信（已完成）
- 🔄 A7670E MQTT上传功能（待开发）

## 🔌 硬件连接

### 1. 开发板引脚
| 引脚 | 功能 | 说明 |
|------|------|------|
| **PB6** | USART1_TX | 连接ST-LINK虚拟COM口，用于调试 |
| **PB7** | USART1_RX | 连接ST-LINK虚拟COM口，用于调试 |
| **PB5** | LPUART1_TX | 连接A7670E模块RXD |
| **PC0** | LPUART1_RX | 连接A7670E模块TXD |
| **GND** | 地 | 共地连接 |

### 2. A7670E模块连接
```
STM32WB5MM          A7670E模块
─────────────────────────────
PB5 (LPUART1_TX) →  RXD
PC0 (LPUART1_RX) ←  TXD
GND              -  GND
                    VBAT (3.8-4.2V供电)
```

**⚠️ 注意事项：**
- A7670E需要独立供电（3.8-4.2V，峰值电流可达2A）
- 串口电平3.3V，可直接连接
- TX/RX需要交叉连接

## 🏗️ 代码架构说明

### 1. 环形缓冲区设计

**为什么需要环形缓冲区？**

在初期测试中，使用单字节直接转发导致数据丢失：
```
发送: AT         → 接收: A    (丢失'T')
发送: AT+CPIN?   → 接收: AP   (丢失后续字符)
```

**问题原因：** 快速连续数据到达时，单字节缓冲区被覆盖。

**解决方案：** 256字节环形缓冲区

```c
// 缓冲区定义
#define BUFFER_SIZE 256
uint8_t usart1_rx_buffer[BUFFER_SIZE];
uint8_t lpuart1_rx_buffer[BUFFER_SIZE];
volatile uint16_t usart1_rx_head = 0;  // 写入位置
volatile uint16_t usart1_rx_tail = 0;  // 读取位置
```

**工作流程：**
```
┌─────────────────────────────────┐
│  中断（极快，<5us）              │
│  接收1字节 → 写入缓冲区 → 退出   │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│  主循环（安全处理）              │
│  从缓冲区读取 → 转发 → 循环处理  │
└─────────────────────────────────┘
```

### 2. 中断回调函数

```c
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1)
    {
        // 将接收到的数据放入环形缓冲区
        usart1_rx_buffer[usart1_rx_head] = usart1_rx_byte;
        usart1_rx_head = (usart1_rx_head + 1) % BUFFER_SIZE;

        // 重新启动接收
        HAL_UART_Receive_IT(&huart1, &usart1_rx_byte, 1);
    }
    else if (huart->Instance == LPUART1)
    {
        // A7670E数据同样处理
        lpuart1_rx_buffer[lpuart1_rx_head] = lpuart1_rx_byte;
        lpuart1_rx_head = (lpuart1_rx_head + 1) % BUFFER_SIZE;

        // 重新启动接收
        HAL_UART_Receive_IT(&hlpuart1, &lpuart1_rx_byte, 1);
    }
}
```

### 3. 主循环处理

```c
while (1)
{
    // 处理USART1接收到的数据（电脑 → A7670E）
    while (usart1_rx_head != usart1_rx_tail)
    {
        uint8_t data = usart1_rx_buffer[usart1_rx_tail];
        usart1_rx_tail = (usart1_rx_tail + 1) % BUFFER_SIZE;
        HAL_UART_Transmit(&hlpuart1, &data, 1, 100);
    }

    // 处理LPUART1接收到的数据（A7670E → 电脑）
    while (lpuart1_rx_head != lpuart1_rx_tail)
    {
        uint8_t data = lpuart1_rx_buffer[lpuart1_rx_tail];
        lpuart1_rx_tail = (lpuart1_rx_tail + 1) % BUFFER_SIZE;
        HAL_UART_Transmit(&huart1, &data, 1, 100);
    }
}
```

### 4. 关键技术点

| 技术点 | 实现方式 | 优势 |
|--------|---------|------|
| 中断接收 | `HAL_UART_Receive_IT` | 不阻塞CPU，响应快 |
| 环形缓冲 | head/tail指针 | 支持连续高速数据 |
| 透传转发 | 主循环轮询缓冲区 | 避免中断嵌套死锁 |
| 看门狗禁用 | 注释初始化代码 | 避免自动复位 |

## 📊 架构演进

### 之前的架构（有线+Python中转）
```
断路器数据 → STM32读取 → JSON封装 → 串口输出
    ↓
Python脚本接收（需要电脑运行）
    ↓
MQTT发布 → EMQX → Node-RED → InfluxDB → Grafana
```

**问题：**
- ⚠️ 需要有线连接
- ⚠️ 需要电脑/树莓派一直运行Python
- ⚠️ 增加故障点和成本

### 目标架构（无线直连）
```
断路器数据 → STM32读取 → JSON封装
    ↓
STM32 → A7670E(4G) → 直接MQTT发布到EMQX
    ↓
EMQX → Node-RED → InfluxDB → Grafana
```

**优势：**
- ✅ 无需Python脚本（省掉中转）
- ✅ 无需有线连接（完全无线）
- ✅ 独立运行（STM32+A7670E）
- ✅ 降低成本（不需要额外设备）

## 🔧 故障排查

### 问题1：串口无响应

**现象：** 串口助手无任何输出

**排查步骤：**
1. 检查ST-LINK连接（USB线）
2. 检查串口助手波特率（115200）
3. 检查代码是否成功烧录
4. 检查看门狗是否已禁用

**解决方案：**
```c
// 确认以下代码已注释
// MX_IWDG_Init();
// MX_WWDG_Init();
```

---

### 问题2：数据丢失（只收到部分字符）

**现象：** `AT` → 只收到 `A`

**原因：** 缓冲区冲突或中断配置问题

**解决方案：** 已通过环形缓冲区解决

**验证方法：**
```bash
发送: AT+CSQ
接收: AT+CSQ\r\n+CSQ: 23,99\r\n\r\nOK\r\n  # 应完整
```

---

### 问题3：发送命令后卡死

**现象：** 发送几个AT命令后系统无响应

**原因：** 中断嵌套死锁（已解决）

**之前的错误代码：**
```c
// ❌ 错误：在中断中使用阻塞发送
HAL_UART_Transmit(&hlpuart1, rxBuffer, 1, 100);
```

**正确代码：**
```c
// ✅ 正确：使用环形缓冲区
usart1_rx_buffer[usart1_rx_head] = usart1_rx_byte;
usart1_rx_head = (usart1_rx_head + 1) % BUFFER_SIZE;
```

---

### 问题4：A7670E无响应

**现象：** 发送AT命令无任何回复

**排查步骤：**
1. 检查A7670E供电（3.8-4.2V，峰值2A）
2. 检查TX/RX是否交叉连接
3. 检查GND是否共地
4. 检查模块是否上电（LED指示灯）

**测试方法：**
```bash
# 最基础测试
AT  # 应该返回 OK

# 如果无响应，尝试复位模块
断电 → 等待5秒 → 上电
```

---

### 问题5：信号强度低

**现象：** `AT+CSQ` 返回 `+CSQ: 5,99` （信号弱）

**排查步骤：**
1. 检查SIM卡是否插好
2. 检查天线是否连接
3. 移动到信号好的位置
4. 检查SIM卡是否欠费

**信号强度参考：**
```
0-9:    很差（可能无法连接）
10-14:  较差
15-19:  一般
20-25:  良好
26-31:  优秀
```
