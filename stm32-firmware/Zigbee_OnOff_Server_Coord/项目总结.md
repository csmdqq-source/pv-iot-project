# STM32WB Zigbee智能断路器远程监控系统 - 项目总结

## 📋 项目概述

本项目实现了一套**完整的智能断路器无线监控与数据上传系统**，通过Zigbee无线网络采集断路器数据，经4G网络上传至云端，最终在可视化平台展示。

**核心特点：**
- ✅ 完全无线化（Zigbee + 4G）
- ✅ 无需中间网关设备
- ✅ 独立运行，实时数据上传
- ✅ 成本低，易部署

## 🏗️ 系统架构详解

### 架构对比

#### 之前架构 (有线+Python)
```
断路器 → STM32 → USB串口 → 电脑/树莓派 → Python脚本 → MQTT → 云端
         (Zigbee)  (有线)    (必须运行)     (中转)
```
**问题：**
- ❌ 需要有线连接
- ❌ 需要额外设备（电脑/树莓派）
- ❌ 增加成本和故障点

#### 当前架构 (完全无线)
```
断路器 → STM32WB → A7670E → MQTT → 云端
         (Zigbee)   (4G)    (直连)
```
**优势：**
- ✅ 完全无线化
- ✅ 独立运行
- ✅ 降低成本
- ✅ 易于部署

### 4. 数据解析
**文件:** `STM32_WPAN/App/app_zigbee.c:1495`

```c
/**
 * 解析功率数据
 * 格式: [电压2字节][电流3字节][功率3字节]
 */
static void ParseTuya_PowerData(const uint8_t *data, uint8_t len)
{
  // 解析电压 (Big Endian, 单位: 0.1V)
  uint16_t voltage_raw = ((uint16_t)data[0] << 8) | data[1];

  // 解析电流 (Big Endian, 单位: mA)
  uint32_t current_ma = ((uint32_t)data[2] << 16) |
                        ((uint32_t)data[3] << 8)  |
                        data[4];

  // 解析功率 (Big Endian, 单位: W)
  uint32_t power_w = ((uint32_t)data[5] << 16) |
                     ((uint32_t)data[6] << 8)  |
                     data[7];

  // 存储到全局变量
  zigbee_app_info.tuya_voltage_dv = voltage_raw;
  zigbee_app_info.tuya_current_ma = current_ma;

  // 打印日志
  APP_DBG("Voltage: %u.%u V, Current: %lu mA, Power: %lu W",
          voltage_raw / 10, voltage_raw % 10,
          (unsigned long)current_ma,
          (unsigned long)power_w);
}
```

### 5. MQTT数据上传
**文件:** `STM32_WPAN/App/app_zigbee.c:1968`

```c
/**
 * MQTT JSON上传
 * 作用: 封装JSON数据并通过A7670E上传
 */
static void APP_ZIGBEE_MqttUploadJson(void)
{
  char json[MQTT_JSON_BUF_SIZE];

  // 获取数据
  uint16_t voltage_raw = zigbee_app_info.tuya_voltage_dv;
  uint32_t current_ma = zigbee_app_info.tuya_current_ma;
  uint32_t power_w = (uint32_t)((voltage_raw / 10) * (current_ma / 1000));
  uint32_t timestamp = APP_ZIGBEE_GetTimestampSeconds();

  // 转换格式
  uint16_t voltage_int = voltage_raw / 10;
  uint16_t voltage_dec = voltage_raw % 10;
  uint32_t current_int = current_ma / 1000;
  uint32_t current_dec = (current_ma % 1000) / 10;

  // 封装JSON
  snprintf(json, sizeof(json),
           "{"
           "\"device_id\":\"%s\","
           "\"timestamp\":%lu,"
           "\"voltage\":%u.%u,\"current\":%lu.%02lu,"
           "\"power\":%lu,"
           "\"status\":%s,"
           "\"efficiency\":0.5,\"temperature\":35.3"
           "}",
           MQTT_DEVICE_ID,
           (unsigned long)timestamp,
           voltage_int, voltage_dec,
           (unsigned long)current_int, (unsigned long)current_dec,
           (unsigned long)power_w,
           zigbee_app_info.tuya_breaker_on ? "true" : "false");

  // 通过A7670E发布
  A7670E_RequestPublishDefault(json, strlen(json));

  APP_DBG("MQTT JSON queued: %s", json);
}
```

### 6. A7670E驱动状态机
**文件:** `Core/Src/a7670e.c`

```c
/**
 * A7670E状态机
 * 状态: INIT → NET_CHECK → NET_REG → MQTT_START → MQTT_CONNECT → READY
 */
void A7670E_Process(void)
{
  switch (current_state) {
    case STATE_INIT:
      // 发送AT测试
      SendAtCommand("AT\r\n");
      break;

    case STATE_NET_CHECK:
      // 检查SIM卡和信号
      SendAtCommand("AT+CPIN?\r\n");
      SendAtCommand("AT+CSQ\r\n");
      break;

    case STATE_NET_REG:
      // 网络注册
      SendAtCommand("AT+CGATT=1\r\n");
      SendAtCommand("AT+CGDCONT=1,\"IP\",\"CMNET\"\r\n");
      break;

    case STATE_MQTT_START:
      // 启动MQTT
      SendAtCommand("AT+CMQTTSTART\r\n");
      break;

    case STATE_MQTT_CONNECT:
      // 连接Broker
      SendAtCommand("AT+CMQTTCONNECT=0,"
                    "\"tcp://broker.emqx.io:1883\",60,1\r\n");
      break;

    case STATE_READY:
      // 就绪，处理发布队列
      if (publish_pending) {
        PublishMessage();
      }
      break;
  }
}
```

### 问题4: MQTT连接失败

**现象:** A7670E网络正常但MQTT连接失败

**解决方案:**
- 确认Broker地址正确
- 检查网络是否限制1883端口
- 尝试使用其他MQTT Broker

## 🎓 技术亮点

### 1. 涂鸦私有协议破解
成功逆向并适配了涂鸦Zigbee私有协议 (0xEF00集群)，实现了对第三方设备的完整控制。

**关键技术:**
- DP命令格式解析
- IEEE地址绑定机制
- 多种Payload格式兼容
- 序列号管理

---

### 2. 完全无线化架构
实现了从设备到云端的完整无线链路，无需任何有线连接和中转设备。

**优势对比:**
| 方案 | 成本 | 部署难度 | 可靠性 | 功耗 |
|------|------|---------|--------|------|
| 有线+Python | 高 | 难 | 中 | 高 |
| **无线直连** | **低** | **易** | **高** | **低** |

---

### 3. 双串口架构
巧妙利用STM32WB的双串口资源，实现调试与通信分离。

**设计优势:**
- USART1专用调试，不影响数据传输
- LPUART1低功耗通信，节省能源
- 环形缓冲区防止数据丢失
- 中断驱动，响应迅速

---

### 4. 状态机驱动
A7670E驱动采用状态机设计，自动化处理网络连接和MQTT发布。

**状态流转:**
```
INIT → NET_CHECK → NET_REG → MQTT_START →
MQTT_CONNECT → READY → PUBLISHING
```

**容错机制:**
- 自动重连 (网络断开)
- 超时重试 (命令失败)
- 队列缓冲 (消息积压)
